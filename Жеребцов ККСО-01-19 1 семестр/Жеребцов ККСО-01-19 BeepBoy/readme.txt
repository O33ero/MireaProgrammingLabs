***Инструкция по установке и эксплуатации BeepBoy.***

/*           Инструкция по установке            */

I) В исходном архиве находится:
	- Руководство по установке и эксплуатации
	- Исходный код программы
	- Файлы функций для языков С, С++, С#
	- Файл сплошной нумерации(ID)
	- Пустая папка с названием c-file
	- Исполняющий файл .exe

II) Установка:
	Все файлы .txt (функции и prev_id) должны находится рядом (в одной директории) с испол-
	няющим файлом. Также программой предусмотренна возможность запись работы датчиков в 
	лог-файл.

	По умолчанию: создайте папку log на диске C: (C:\\log\\). В этой папке будет создан 
	лог-файл, в который будет записыватся работа датчиков.

	Для изменения или разделения лог-файлов, измените адрес записи в каждом из файлов функ-
	ций.

	В пустую папку c-files помещаются файлы исходного кода, которые необходимо изменить.

	Что надо сделать:
	 - Проверить наличие всех файлов согласно списку
	 - Создать папку log на диске C: (C:\\log\\)
	 - Поместить файлы в папку c-file


	В случае успешной установке, программа изменить исходный файл и сообщит о добавлении
	нового BB в консоль.


/*              Общий принцип работы               */
1) Ключевые понятия
		BeepBoy - программа, написанная на языке С, по внедрению датчика в исходный код про-
	граммы на языке C, C++ и C#. Beep(звук, который издает датчик при срабатывании) Boy(ма-
	ленький мальчик) для простоты понимания далее заменяется на BB. Все файлы и функции также
	имеют в названии Beep или BB.

2) Общий принцип работы
	BB открывает файл в формате .txt и расматривает его посимвольно, запоминая последние сло-
	ва. Анализируя последовательность символов и слов, BB понимает, в какой части программы 
	он находится и из этих соображений добавляет:
		a) функцию, которая будет записывать свою работу в лог-файл
		б) нового BB, робота который добавляется в каждую функцию и
		   сообщает о своей работе.

	В BB предусмотренно сплошное нумерование. Т.е. программа может сработать несколько раз, и
	каждый раз у всех BB будет различный ID.

	Также после первого запуска, BB создаст файл .csv - база данных всех добавленных BB. В ней
	хранится ID, функция и программа, в которую добавлен BB.

3) Ложное срабатывание и нереализованные функции
		По условия задания, BB должен уметь обрабатвать создание потоков и работу функций из по-
	токов. Но принцип работы BB не позволяет обрабатывать функции из потоков (если это и возмо-
	жно, то принцип обработки сверхсложен и необходимо изменять всю программу).

		Также BB не различает, есть ли уже в программе функция BB. Поэтому файлы, которые уже 
	были обработаны, могут быть обработаны еще раз. В следствии будут добавлены новые BB, кото-
	рые дублируют уже ранее добавленные.

		Один из способ защитить функцию от внедрения BB - многострочный комментарий между
	закрывыющей скобкой(')') и открывающей фигурной скобкой ('{'). Либо ложные скобки в одностро-
	чном комментарии.
		Также возможны другие манипуляции с комментариями для ложного срабатывания.


	Замечание от автора: В некоторых местах, где программа должна ошибочно добавлять бота,
	(например, в строку for(///){ }) BB, по непонятной автору причине, этого не делает, поэтому не
	возможно обработать эту ошибку
